PRODUCTION is a cloud-based project managment application for writers, producer and all manner of magic makers

Problem

PRODUCTION makes it easy for artists and producers to collaborate by supporting all major, industry-standard file types including .fdx .cad, and music notation software.

Solution

Research

User Survey

The first thing I did was to create a survey to figure out what other cloud computing computing applications creative types used and how they used them. Below are my key findings.

49 Total Respondants

Over three-quarters of the have taken part in a collaborative process.

95.7% of respondents are google drive users

Most of them use cloud drives for organization, collaboration, remote file storage

Hardly any use cloud computing apps for content creation.

User Personas

The survey respondents listed a variety of creative rolls. To limit the scope of work I decided to focus on specifically writers and producers and the script development phase as it is generally the first phase of any production.

Branding

To make a piece of art is hard. Its natural state is to exist inside the maker's head. Always perfect but never seen by anyone else. It only takes one person to come up with an idea but it takes a team of people constantly communicating and encouraging each other to make that idea a reality. That’s why I designed “PRODUCTION”

To produce. To make something. PRODUCTION should not just encourage creativity but also add a tangibility to the work. Adding voices and collaboration to the process creates a palpable sense of urgency that helps propel an idea to a draft to a production-ready script.

Various dashboard ideas and some fun skeuomorphic script and note processing

Logo

As for the logo. I explored a lot of different ideas around the idea of growing something from the mind. There are a few crude drawings of brains or brains becoming butterflies. Those… didn’t work. My favorite is the lightbulb. Lighting is obviously a major feature of both film and theater production (aka our target users).

To come up with the name I created a list. First, of different production documents, I found that to be fairly cold. It may work for production administration but it would leave the creatives out. (Not that producers and administrators aren’t creative). Then I started thinking about creativity terms that implied growing something. Things like spark and cocoon. I actually settled on “PRODUCTION” because it satisfies both the administrative and creative users. Production isn’t a destination. It’s a process. It’s where plans are made and executed.

Colors

As for colors. I wanted a cozy feeling. I settled on a burgundy and tan color scheme. To me, it felt like curling up next to the fire.

This is a later iteration. User testing and feedback proved that I had made a mistake with the tans and green. Here I added a blue to really give it a nice pop as well continue to foster creativity.

Information Architecture

User Flows

The biggest challenge regarding the user flows came from having to explain the app.

You’ll undoubtedly notice that there are no mobile user flows. That is intentional. Long-form dramatic writing is still in the realms of keyboards and cursors. This doesn’t mean that mobile usability could not be implemented in the future. However, to solve the problem and to create a minimum viable product I stuck to a desktop app.

More Testing

User Testing

I did three test asking the users to perform the following tasks

Sign up for an Account

Signing up for an account went well. The call-to-action buttons did their job. The teal color had people gravitate right towards it.

Create a File

Creating a file went similarly. The user flow from login to document creation did its job well.

Share a File

Logging in to get to sharing was rough. Multiple testers had trouble finding the login bar. I fixed this by making the login bar static.

The biggest takeaway from testing was that I wished I’d done a round sooner. It would have saved me a lot of antacids if I’d just tested the original design and gotten people’s thoughts on it sooner rather than waiting for official grading team feedback. Oh well, we live to learn, right?

Preference Testing

I was getting some great feedback through my design process. However, as with all feedback, there were some contridictions. I moved into preference testing to try to resolve those contridictions.

I tested the iconography and the hero image. There was a pretty clear winner in all but the header image option. However, both the icon sets went the way I thought they would go so I just went with my instinct.

Conclusion
